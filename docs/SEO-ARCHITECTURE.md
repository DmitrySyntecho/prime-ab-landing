## 9. INTERNAL LINKING STRATEGY

### Cross-Linking Rules

**Service Pages → Industry Pages**
- Each service page links to relevant industries that use that service
- Example: LED Wall Rental → Fashion Shows, Trade Shows, Brand Activations, Pop-Up Events

**Service Pages → Location Pages**
- Each service page links to major city pages
- Example: AV Production → Miami, Los Angeles, New York, Las Vegas

**Service Pages → Resource Pages**
- Each service page links to 3-5 relevant educational resources
- Prioritize resources that answer "how much does this cost" and "how does this work"

**Industry Pages → Service Pages**
- Each industry page links to all relevant services (high priority)
- Example: Fashion Shows → Lighting, Staging, Video Production

**Industry Pages → Location Pages**
- Industry hub pages link to top city pages for that vertical

**Industry Pages → Package Pages**
- Link to packages designed for that industry (2-3 most relevant)

**Location Pages → Service Pages**
- Link to all services available in that location

**Location Pages → Industry Pages**
- Link to industries active in that city/region

**Location Pages → Resource Pages**
- Link to local resources and city-specific guides

**Resource Pages → Service Pages**
- Educational content links to related services (primary CTA)
- Example: "LED Wall Size Guide" → LED Wall Rental service

**Resource Pages → Industry Pages**
- Link to industries mentioned in resource

**Resource Pages → Equipment Pages**
- Link to equipment discussed in resource

**Comparison Pages → Service Pages**
- Strong links to recommended services
- CTA link to service consultation

**Comparison Pages → Package Pages**
- Link to packages that solve the comparison problem

**Equipment Pages → Service Pages**
- High priority links to services using that equipment

**Equipment Pages → Package Pages**
- Link to packages including that equipment

**Package Pages → Service Pages**
- Detailed breakdown of services included with links

**Package Pages → Equipment Pages**
- Show equipment included in package

**Package Pages → Location Pages**
- Link to availability in major cities

**Tool Pages → Service Pages**
- Link to related services based on tool results

**Tool Pages → Package Pages**
- Link to packages matching tool output

## 13. URL STRUCTURE RULES

**ALWAYS USE:**
- Hyphens: `/led-wall-rental` (NOT `/ledwallrental`)
- Lowercase: `/services/av-production` (NOT `/Services/AV-Production`)
- No trailing slashes: `/services/led-wall-rental` (NOT `/services/led-wall-rental/`)
- Descriptive slugs: `/services/conference-av-services` (NOT `/services/s1`)
- Consistent nesting: `/locations/{state}/{city}` pattern

**NEVER USE:**
- Underscores
- Numbers as identifiers
- Dynamic parameters in slugs
- Duplicate content across URLs

## SUMMARY

This architecture transforms Prime Line AV into a **national SEO traffic engine** by:

1. **Capturing Commercial Intent** - Service pages target high-converting equipment rental keywords
2. **Ranking Locally** - 50+ state pages + 50 city pages capture local search intent
3. **Winning AI Search** - Resource and comparison pages target AI search queries
4. **Converting Equipment Searches** - Cross-linking paths from equipment → full service projects
5. **Supporting Packages** - Pre-configured solutions reduce buyer decision paralysis
6. **Enabling Scalability** - Architecture supports 200+ pages with minimal content duplication

The internal linking strategy ensures every page supports adjacent pages, creating a cohesive SEO structure where traffic flows from awareness (resources) → consideration (industries, comparisons) → decision (packages) → conversion (services).
