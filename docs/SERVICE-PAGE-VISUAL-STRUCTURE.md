## Responsive Breakpoints

### Mobile (Mobile-First)
- Equipment grid: 1 column
- Industries: 2 columns
- Cities: 3 columns
- Resources: 1 column (stacked)
- Packages: 1 column (stacked)
- Process: Single column timeline (numbered steps)
- FAQ: Full-width accordion

### Tablet (md: 768px)
- Equipment grid: 2-3 columns
- Industries: 3-4 columns
- Cities: 5 columns
- Resources: 2-3 columns
- Packages: 2 columns
- Process: Side-by-side (number + content)
- FAQ: 2-column layout (optional)

### Desktop (lg: 1024px+)
- Equipment grid: 3 columns
- Industries: 4-5 columns
- Cities: 5 columns
- Resources: 3 columns
- Packages: 4 columns
- Process: Full timeline with visual flow
- FAQ: Full-width accordion

## Color Scheme (Design System)

**Primary Colors:**
- Dark Background: `#0B1217` (hero, footer CTA, dark sections)
- Accent Green: `#4ADE80` (buttons, links, highlights)
- Accent Green (Hover): `#22c55e` (darker green on interaction)
- Branded Green: `#16A34A` (secondary accent for links)

**Neutral Colors:**
- White: `#FFFFFF` (primary text, backgrounds)
- Light Gray: `#F3F4F6` (alternate section backgrounds)
- Medium Gray: `#9CA3AF` (secondary text, labels)
- Dark Gray: `#1F2937` (primary headings)
- Very Light Gray: `#E5E7EB` (borders, dividers)

**Backgrounds:**
- Odd Sections: White (`#FFFFFF`)
- Even Sections: Gray (`#F9FAFB` or `#F3F4F6`)
- Hero Section: Dark with gradient overlay

---

## Typography Hierarchy

**Headings:**
- H1 (Hero): 48-56px, font-black, tracking-tight
- H2 (Section): 32-40px, font-bold
- H3 (Card title): 18-20px, font-semibold

**Body:**
- Paragraph: 16px, line-height 1.5-1.6, text-gray-600
- Small text (labels): 12-14px, uppercase, tracking-wider
- Card text: 14-16px

**Interactive:**
- Button text: 16px, font-semibold, all-caps labels on hover
- Link text: 14-16px, underline on hover, green color

---

## Component Interactions

**Cards on Hover:**
- Border color: Gray → Green (#16A34A)
- Shadow: None → md (0 4px 6px -1px rgba(0,0,0,0.1))
- Background: Static or subtle color shift

**Buttons:**
- Primary (Green): bg-[#4ADE80] text-[#0B1217] → hover:bg-[#22c55e]
- Secondary (White): border-2 border-white/30 text-white → hover:bg-white/10
- Rounded: 6px (md class)
- Padding: px-8 py-4 (large), px-6 py-3 (medium)

**Form Elements:**
- Quote form opens as modal overlay
- Sticky CTA appears on scroll past fold

## Internal Linking Strategy

**SEO Clusters by Section:**

1. **Equipment Section**
   - Links to: `/equipment/[category]/[product]`
   - Anchor text: Equipment names, specs
   - Purpose: Establish equipment authority

2. **Industries Section**
   - Links to: `/industries/[slug]`
   - Anchor text: Industry names
   - Purpose: Show service applicability across verticals

3. **Locations Section**
   - Links to: `/locations/[state]`
   - Anchor text: State names
   - Purpose: Geographic authority

4. **Resources Section**
   - Links to: `/resources/[slug]`
   - Anchor text: Guide titles
   - Purpose: Content hub interconnection

5. **Comparisons Section**
   - Links to: `/comparisons/[slug]`
   - Anchor text: Comparison titles
   - Purpose: Competitive positioning

6. **Packages Section**
   - Links to: `/packages/[slug]`
   - Anchor text: Package names
   - Purpose: Drive pre-configured solutions

7. **Related Services Section**
   - Links to: `/services/[slug]`
   - Anchor text: Service names
   - Purpose: Cross-sell complementary services

## Mobile-First Layout Principles

✓ **Stack Everything** - Sections stack vertically on mobile
✓ **Full Width** - Use max-w-7xl with px-4 for breathing room
✓ **Tap Targets** - Buttons/links minimum 44x44px
✓ **Readable Text** - 16px minimum on mobile, leading-relaxed
✓ **Image Optimization** - Use Next/Image with responsive sizes
✓ **Sticky Elements** - CTAs stick to bottom on mobile

---

## Accessibility Features

✓ Semantic HTML (section, header, nav elements)
✓ Breadcrumb navigation for context
✓ Heading hierarchy (h1 → h2 → h3)
✓ Color contrast meets WCAG AA standards
✓ Buttons have clear focus states
✓ Form labels associated with inputs
✓ Accordion items have proper roles/aria attributes

---

## Performance Optimizations

✓ Image lazy loading (Next/Image component)
✓ Code splitting by route (Next.js dynamic routes)
✓ Component memoization for CTAs
✓ CSS class consolidation (Tailwind utility classes)
✓ External scripts (Vidalytics) loaded after page interactive
✓ Sticky CTA conditionally rendered only when needed
